# Тест-кейсы приложения «Best Offer Finder»

| ID | Заголовок | Предусловие |
|----|-----------|-------------|
| TC-01 | Загрузка списка объявлений | Открыта главная страница (`http://localhost:5173`) |
| TC-02 | Отображение цены в карточке | Главная страница загружена |
| TC-03 | Загрузка изображения товара | Главная страница загружена |
| TC-04 | Переход на объявление на FarPost | Карточка объявлена на главной странице |
| TC-05 | Переключение на вкладку «Лента» | Открыта другая вкладка (напр., «Категории») |
| TC-06 | Переключение на вкладку «Категории» | Открыта вкладка «Лента» |
| TC-07 | Получение данных через API | Back-end запущен |
| TC-08 | Обработка ошибки парсинга | Интернет отключён или FarPost недоступен |
| TC-09 | Адаптивность интерфейса | Открыта главная страница |
| TC-10 | Безопасная обработка URL изображений |

---

## Подробные тест-кейсы

### TC-01: Загрузка списка объявлений

| Шаг | Ожидаемый результат |
|-----|---------------------|
| 1. Открыть `http://localhost:5173` | Страница загружается без ошибок |
| 2. Дождаться завершения загрузки | Отображается ≥1 карточка с iPhone |
| 3. Проверить первую карточку | Есть заголовок, цена (число), изображение |

---

### TC-02: Отображение цены в карточке

| Шаг | Ожидаемый результат |
|-----|---------------------|
| 1. Найти любую карточку | Цена отображается как `5000`, `31999` и т.д. |
| 2. Проверить формат | Только цифры, без пробелов, запятых |

---

### TC-03: Загрузка изображения товара

| Шаг | Ожидаемый результат |
|-----|---------------------|
| 1. Найти карточку с реальным фото | Изображение загружено и отображается корректно |

---

### TC-04: Переход на объявление на FarPost

| Шаг | Ожидаемый результат |
|-----|---------------------|
| 1. Кликнуть по кнопке перейти | Открывается новая вкладка |
| 2. Проверить URL новой вкладки | Начинается с `https://www.farpost.ru/` |

---

### TC-05: Переключение на вкладку «Лента»

| Шаг | Ожидаемый результат |
|-----|---------------------|
| 1. Нажать кнопку «Лента» | Отображается список объявлений |
| 2. Проверить выделение кнопки | Кнопка «Лента» визуально активна |

---

### TC-06: Переключение на вкладку «Категории»

| Шаг | Ожидаемый результат |
|-----|---------------------|
| 1. Нажать кнопку «Категории» | Интерфейс переключается на экран категорий |
| 2. Проверить содержимое | Отображаются 4 категории: *Электроника*, *Телефоны*, *Авто*, *Дом* |

---

### TC-07: Получение данных через API

| Шаг | Ожидаемый результат |
|-----|---------------------|
| 1. Открыть `http://localhost:8000/api/items` | Возвращается JSON |
| 2. Проверить структуру | Есть поле `"items"`, тип — массив, каждый элемент содержит `title`, `price`, `url`, `image_url`, `description` |

---

### TC-08: Обработка ошибки парсинга

| Шаг | Ожидаемый результат |
|-----|---------------------|
| 1. Запустить сервер без интернета | Сервер запускается, не падает |
| 2. Открыть фронтенд | Отображается пустой список или сообщение «Нет данных» |

---

### TC-09: Адаптивность интерфейса

| Шаг | Ожидаемый результат |
|-----|---------------------|
| 1. В DevTools выбрать iPhone SE (375×667) | Карточки и кнопки навигации корректно размещены |
| 2. Прокрутить список | Нет горизонтального скролла, текст не обрезан |

---

### TC-10: Безопасная обработка URL изображений

| Шаг | Ожидаемый результат |
|-----|---------------------|
| 1. Вручную подставить `image_url = "javascript:alert(1)"` | Изображение не загружается |
| 2. Проверить `<img src="...">` | Атрибут `src` заменён на placeholder, JS не выполняется |
